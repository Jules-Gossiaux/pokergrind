# Feuille de route

Cette feuille de route décrit l'ordre de travail, pas des dates de livraison.

## Étape 0 — Socle

- documentation initiale ;
- projet Android et automatisation de build ;
- thème sombre et composants fondamentaux ;
- stockage local et navigation ;
- format structuré et validation des ranges.

## Étape 1 — Open BTN vertical

- transcription et validation de la range BTN ;
- session d'entraînement Open / Fold ;
- feedback immédiat ;
- matrice `Voir la range` ;
- historique local minimal ;
- tests du parcours complet.

Cette tranche doit fonctionner de bout en bout avant l'ajout d'un deuxième
spot.

## Étape 2 — Parcours des open ranges

- CO, HJ, UTG et SB ;
- déblocage pédagogique ;
- maîtrise sur les 30 dernières réponses ;
- couverture et diversité ;
- progression par spot.

## Étape 3 — Boucle quotidienne

- répétition espacée ;
- session quotidienne de 20 questions ;
- entraînement libre ;
- XP, statistiques et série quotidienne ;
- reprise d'une session interrompue.

## Étape 4 — Stabilisation MVP

- tests sur appareils Android ciblés ;
- accessibilité et finitions ;
- optimisation des performances ;
- sauvegarde et migration des données locales ;
- génération d'un APK signé ;
- documentation d'installation.

## Après le MVP

Dans l'ordre prévu :

1. défenses de BB ;
2. 3-bets ;
3. défense contre 3-bet ;
4. récapitulatif préflop ;
5. autres fondations du poker.
